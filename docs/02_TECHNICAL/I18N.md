# I18N

## Idiomas

La app soporta:

- `es` - español, idioma por defecto.
- `ca` - catalán.
- `en` - inglés.

## Implementación

- Configuración: `src/i18n/i18nConfig.js`.
- Provider: `src/i18n/I18nProvider.jsx`.
- Diccionarios UI: `src/i18n/locales/es.js`, `ca.js`, `en.js`.
- Diccionarios legales: `src/i18n/legal/es.js`, `ca.js`, `en.js`.

## Persistencia

- Key de `localStorage`: `nensgo.language`.
- Si `localStorage` falla, la app mantiene el cambio en memoria.
- `<html lang>` se actualiza con el idioma activo.

## Alcance

Se traduce:

- copy estático de UI;
- textos de navegación;
- estados vacíos/errores controlados;
- páginas legales mediante bundle lazy.

No se traduce:

- `NensGo`;
- nombres de actividades;
- nombres de centros;
- nombres de ciudades o municipios;
- nombres de personas;
- direcciones;
- emails;
- URLs;
- valores de contacto;
- contenido dinámico de Supabase.

## Cómo añadir una key

1. Añadir la key en los tres diccionarios de `src/i18n/locales`.
2. Si es legal, añadirla en los tres archivos de `src/i18n/legal`.
3. Usar `const { t } = useI18n()` en el componente.
4. Llamar `t("ruta.de.key")`.
5. Evitar traducir marca, nombres propios o datos dinámicos.

## Limitaciones actuales

- No hay rutas por idioma.
- No hay `hreflang`.
- No hay traducción dinámica de actividades.
- No hay fallback editorial para contenido parcial por idioma.
